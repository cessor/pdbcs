Navigation

c or continue Resume execution of the program to next breakpoint.
step Resume execution to the next line or function call.
Step will also "Step Into" called functions and stop inside.
next Resume execution to the next line.
Next will skip over called functions and stop after them.
down Move down one frame.
down count Move down more frames.
up Move up one frame.
up count Move up more frames.
until Resume execution until a line after this one is reached. Stop when the current frame returns.
until linenumber Resume execution until the line after the specified one is reached. Stop when the current frame returns.
return Resume execution until the current function returns.
jump lineno In the bottom most frame, set the next line to execute.
Helps to re-execute or skip some code.
You can't jump into a for-loop or out of a finally clause.


Intervention

interact Start an interpreter with all the local and global variables in the current scope.
statement Execute the statement.
! statement Prefix with ! if the first word is a debugger command.
global statement Set a global variable.
run [args ...],
restart [args ...]		 Restart the running program with these arguments.
Breakpoints are preserved.
alias name command Create a new command with the given name.
A command can be any legal program.
Aliases can override debugger commands.
To get them back, remove the alias.
alias List existing aliases.
alias name Inspect given alias.
unalias name Delete this alias.


Inspection: Code

list . Print 11 lines of code around the current line.
list Continues printing.
If it has not yet printed, it prints 11 lines around the current line.
list linenumber Print 11 lines around that line.
list first last Print this range.
list line count Print this range.
longlist (ll) List the complete code for this function or frame.
source object Try to display source code for this object.
An object must be: module, class, method, function,
traceback, frame, or code object


Inspection: Data

args Print the argument list of the current function
p expression Evaluate the expression and print its value.
pp expression Pretty print it.
print(...) An actual print.
whatis object Print the type of the expression.
display List all display expressions.
display expression Display the expression's value, if it changed since the last execution,
undisplay Clear all display expressions.
undisplay expression Remove this display expression.


Inspection: Breakpoints

Adding and Removing Breakpoints

break Lists all breakpoints.
Returns Format: Break, Breakpoints, Hit Count, Ignore Count, Condition
break linenumber Break there.
break file:linenumber Break there, but in a different file.
break function Break at first executable statement in function.
break location, condition Break there if condition is met, but remove breakpoint after hitting it.
tbreak location
[,condition]		 Break there if condition is met, but remove breakpoint after hitting it.
Syntax is the same as for break.
clear Remove all breakpoints
clear location Remove all specific breakpoints at this position
clear breakpoints Remove all specific breakpoints

Toggling Breakpoints

disable Disable all breakpoints.
(Note: This is a guess)
disable breakpoints Don't break there.
Specify list of breakpoint numbers
enable Reenable all breakpoints.
(Note: This is a guess)
enable breakpoints Reenable disabled breakpoints.
ignore breakpoint Ignore breakpoint
ignore breakpoint count Ignore breakpoint so many times.
The ignore count is decremented upon hitting the breakpoint.

Altering Breakpoints

condition breakpoint Remove all conditions from this breakpoint.
condition breakpoint condition Install a new condition.
commands breakpoint Prompts for a list of commands for the breakpoint.
For example, you can specify to print a local variable. Navigation commands will resume execution of the program.
commands breakpoint
-> (com) end;		 Specifying end will clear the commands list.


Orientation

help Print available commands.
help command Print help about a command.
help pdb Print docstring documentation of debugging module.
help exec Print help about a command.
where Print a stacktrace.
The current frame is at the bottom.
An arrow indicates the current frame.
Commands are executed against this frame as their context.
quit Guess what this does.

Symbols

position Format: [filename:]lineno
Break there.
function Break at first executable statement in function.
location Format: position or function
condition Only break there if this expression evaluates to True.
breakpoint Format: bpnumber
The breakpoint number
breakpoints Format: bpnumber [bpnumber ...]
List of breakpoint numbers.

Notation and Notes

command In the debugger prompt, you can write this command.
The bold part is the shortcut alias.
(Pdb) Debugger command prompt.
(com) Breakpoint commands prompt.
-> The current line in the current frame.
>> Original line where exception was raised or propagated.
Aliases: Aliases may be nested.
Replaceable parameters can be indicated by %1, %2,
and so on, while %* is replaced by all parameters.

Notation for break and tbreak

break [position [, condition]]
Parameters for breakpoints are location or function, and an optional condition.
Set a breakpoint there, but only if this condition is met.
